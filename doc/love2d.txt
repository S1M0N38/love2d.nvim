================================================================================
INTRODUCTION                                                          *love2d*

love2d.nvim is a plugin to help you develop games with LÖVE.

Table of contents:

1. SETUP: Install and configure the plugin.                     |love2d-setup|
2. COMMANDS: Run, watch, stop your game.                     |love2d-commands|
3. OUTPUT: Floating output panel and diagnostics.              |love2d-output|
4. LSP: LÖVE-aware lua_ls configuration.                          |love2d-lsp|
5. EVENTS: User autocmds for project enter/leave.              |love2d-events|
6. COMPILER: :make integration for LÖVE.                     |love2d-compiler|
7. GAME SETUP: Configure your LÖVE game for the plugin.     |love2d-gamesetup|
8. GLSL: Treesitter injections for inline shaders.               |love2d-glsl|
9. HEALTH: Diagnose issues with :checkhealth.                  |love2d-health|

SETUP                                                           *love2d-setup*

Install love2d.nvim with your preferred method.

Using `vim.pack`: >lua

  vim.pack.add({
    {
      src = 'https://github.com/S1M0N38/love2d.nvim',
      version = vim.version.range('3'),
    },
  })
  require('love2d').setup({})
  vim.keymap.set('n', '<leader>vr', '<cmd>Love run<cr>',    { desc = 'Run LÖVE' })
  vim.keymap.set('n', '<leader>vw', '<cmd>Love watch<cr>',  { desc = 'Watch LÖVE' })
  vim.keymap.set('n', '<leader>vi', '<cmd>Love info<cr>',   { desc = 'Info LÖVE' })
  vim.keymap.set('n', '<leader>vs', '<cmd>Love stop<cr>',   { desc = 'Stop LÖVE' })
  vim.keymap.set('n', '<leader>vo', '<cmd>Love output<cr>', { desc = 'Output panel' })
<

Using `lazy.nvim`: >lua

  {
    'S1M0N38/love2d.nvim',
    version = '3.*',
    event = 'VeryLazy',
    opts = {},
    keys = {
      { '<leader>v',                          desc = 'Run LÖVE' },
      { '<leader>vr', '<cmd>Love run<cr>',    desc = 'Run LÖVE' },
      { '<leader>vw', '<cmd>Love watch<cr>',  desc = 'Watch LÖVE' },
      { '<leader>vi', '<cmd>Love info<cr>',   desc = 'Info LÖVE' },
      { '<leader>vs', '<cmd>Love stop<cr>',   desc = 'Stop LÖVE' },
      { '<leader>vo', '<cmd>Love output<cr>', desc = 'Output panel' },
    },
  }
<

`lazy.nvim` calls |love2d.setup()| automatically with the `opts` table but you
still need to pass an empty table `{}` to the `opts` argument.

The options are merged with the defaults:

>lua
  {
    path_to_love_bin = "love",
    output = nil,
    lsp = true,
  }
<
                                                              *love2d.setup()*
love2d.setup({opts}) ~


  `lsp` (boolean)  Enable automatic lua_ls configuration for LÖVE projects.
    Defaults to `true`. Set to `false` to disable LSP integration entirely
    — useful if you want to provide your own lua_ls config or use a
    different language server.

    When `false`, the plugin does not call |vim.lsp.config()|, |vim.lsp.enable()|,
    or register any LSP autocmds. You are responsible for configuring your
    language server and making the bundled type definitions available
    (they remain on |'runtimepath'| for manual use).

  `path_to_love_bin` (string)  Path to the LÖVE binary. Defaults to `"love"`,
    which works if `love` is in your `PATH`. On macOS, if you installed LÖVE
    as an `.app`, set it to something like
    `"/Applications/love.app/Contents/MacOS/love"`.

  `output` (nil|false|table)   Controls the floating output panel (see
    |love2d-output|).
      • `nil` (default): auto-open the panel.
      • `false`: disable auto-open. Diagnostics still appear inline.
      • `table`: auto-open with custom window config, merged with the
        defaults. Accepts the same keys as the `config` parameter of
        |nvim_open_win()|. For example, a right split instead of a float:

    For example if you prefer a vertical split:
>lua
    {
      output = {
        vertical = true,
        split = "right",
      }
    }
<

================================================================================
COMMANDS                                                     *love2d-commands*

All commands are accessed through the `:Love` command with tab-completed
subcommands:

:Love run ~
  Run the detected LÖVE project. Shows a warning if no project is found or if
  a game is already running.

:Love watch ~
  Run the project and automatically restart it every time you save a Lua file
  (debounced 300ms). Stops any previously running process first.

:Love stop ~
  Stop the running LÖVE process. Also stops watch mode if active.

:Love info ~
  Show the detected project name, entry point, status (running / watching /
  stopped), and last exit code.

:Love output ~
  Cycle the output panel: hidden → focused → hidden. See |love2d-output|.


================================================================================
OUTPUT                                                         *love2d-output*

When a LÖVE project is running, a scratch buffer displays its stdout and
stderr output.

Stderr errors are also shown as inline |vim.diagnostic|
(highlighted like lsp errors) in the corresponding source file.
Note: this requires a custom `love.errorhandler` — see |love2d-gamesetup|.

The output panel has three states:

  hidden → unfocused → focused → hidden

  • hidden:    window is closed.
  • unfocused: window is open, cursor stays in your code.
  • focused:   cursor is inside the output panel.

By default the panel auto-opens unfocused on first output. If you close it
manually it stays closed until the next `:Love run` or `:Love watch`.

Buffer-local keymaps (active inside the output panel):

  `q`        Close the panel.
  `<CR>`     Jump to the file and line under the cursor.

The panel auto-scrolls to the bottom. If you scroll up, it stops scrolling
until you scroll back down.

See |love2d-setup| for how to customize or disable the output panel.

LSP                                                               *love2d-lsp*

When a LÖVE project is detected (see |love2d-events|), the plugin
automatically configures `lua_ls` using |vim.lsp.config|. On project enter it
merges LÖVE-specific settings into your existing `lua_ls` config:

  • LuaJIT runtime
  • LÖVE + LuaSocket type definitions (bundled with the plugin)
  • Disables `duplicate-set-field` and `checkThirdParty` diagnostics

On project leave the LÖVE library paths are stripped and any running `lua_ls`
is notified via `workspace/didChangeConfiguration` so it updates
immediately.

Project detection walks up from the current directory and looks for:

  • `conf.lua` containing `function love.conf`
  • `main.lua` with LÖVE callbacks or `love.*` module usage
  • Any `*.lua` file with `function love.X` callback definitions

No configuration is needed — just open a LÖVE project and `lua_ls` gets the
right completions. For example, press `K` over `love.graphics` to see its
documentation.

EVENTS                                                         *love2d-events*

The plugin fires User autocmds when a LÖVE project is detected or left.
You can use them to run your own code on project enter/leave:

>lua
  vim.api.nvim_create_autocmd("User", {
    pattern = "LoveProjectEnter",
    callback = function(args)
      local root = args.data.path_to_love2d_project
      local main = args.data.path_to_main_lua
      print("LÖVE project at: " .. root)
    end,
  })
<
                                                            *LoveProjectEnter*
LoveProjectEnter ~
  Fired when a LÖVE project is detected. `args.data` contains:
    • `path_to_love2d_project` (string): absolute path to the project root.
    • `path_to_main_lua` (string): absolute path to `main.lua`.

                                                            *LoveProjectLeave*
LoveProjectLeave ~
  Fired when leaving a LÖVE project. `args.data` is a table with
  `path_to_love2d_project` and `path_to_main_lua` both set to `nil`.

Detection triggers on |VimEnter|, |DirChanged|, and |BufEnter|.

COMPILER                                                     *love2d-compiler*

Vim's |:make| command runs a program, captures its output, and populates the
|quickfix| list so you can jump to errors. A *compiler plugin* sets the
|'makeprg'| (what to run) and |'errorformat'| (how to parse errors) for a
specific tool.

This plugin ships a compiler for LÖVE. To use it:

>vim
  :compiler love
<

Now |:make| runs the LÖVE binary on the detected source directory. Errors
matching `file:line: error: message` are parsed into the quickfix list so you
can jump to them with |:copen| and |:cfirst|.

The compiler reads the binary path from |love2d.setup()| and the source path
from the detected project.

Note: LÖVE's default `love.errorhandler` opens a graphical window and never
  exits, which hangs `:make`. You need a custom error handler that writes
  parseable errors to stderr and exits. See |love2d-gamesetup|.

================================================================================
GAME SETUP                                                  *love2d-gamesetup*

By default, LÖVE's error handler opens a graphical window and never exits.
This prevents the output panel and `:make` from capturing errors.

Add the following to your project's `conf.lua` to write errors to stderr
and exit immediately:

>lua
  love.errorhandler = function(msg)
    msg = tostring(msg)
    local file, line, message = msg:match("(%S+):(%d+):%s+(.*)")
    if file and line then
      io.stderr:write(file .. ":" .. line .. ": error: " .. message:gsub("\n.*", "") .. "\n")
    else
      local first_line = msg:match("^(.-)\n") or msg
      io.stderr:write(first_line .. "\n")
    end
    io.stderr:flush()
    os.exit(1)
  end
<

With this in place, runtime errors appear in the output panel and as inline
diagnostics (see |love2d-output|), and `:make` populates the quickfix list
(see |love2d-compiler|). The drawback is that the game will exit immediately
and you will not be presented with the classic LÖVE blue screen error.

BTW if you are reading this, you have descended into the rabbit hole of
terminal-only development. Having errors directly in the terminal is better
for you and your coding agent, so IMHO a custom error handler is the way
to go.

GLSL                                                             *love2d-glsl*

OpenGL Shading Language (GLSL) is a high-level shading language for writing
shaders. See https://www.love2d.org/wiki/love.graphics.newShader section
"Shader Language" for LÖVE-specific aliases.

This plugin provides Treesitter injections so that inline GLSL inside
`love.graphics.newShader()` is recognized as `glsl` code, giving you syntax
highlighting and other Treesitter-based features.

Install the parsers with `:TSInstall lua glsl` (requires nvim-treesitter).
Then reload the buffer with |:edit| to see the shader below highlighted:

>lua
  love.graphics.newShader([[
    vec4 effect(vec4 color, Image tex, vec2 texture_coords, vec2 screen_coords)
    {
        vec4 texturecolor = VideoTexel(texture_coords);
        return texturecolor * color;
    }
  ]])
<

HEALTH                                                         *love2d-health*

Run |:checkhealth love2d| to diagnose common issues. It checks plugin setup,
Neovim version (≥ 0.12.2), LÖVE binary, lua_ls, type definitions,
Treesitter parsers, and GLSL injection.
